HTML

Start with coding up a single question and answer - then use flexboxes to arrange them. At this point using flexboxes for this should be almost second nature. You don‚Äôt need any of those fancy grid frameworks.

One thing you may have noticed is that when you select an answer it is a exclusive selection. There is an input type that already does that automatically. Yup, radio boxes!

But the default styling of radio boxes is so ugly! Sure, but you can do something similar to the checkbox hack and hide the actual radio box but use it for some display logic.

Here is the basic syntax for a radio button with an image label.

<label for="puppy_question"><img src="http://onlines/someimage.png" /></label>
<input type="radio" id="puppy_question" value="puppy"/>

#puppy_question {
display: none;
}

You can actually nest the input inside of the label and then you don‚Äôt have to do the for="elementID" part of the label, it will just automatically serve as the label for the input inside it.

<label>
<img src="http://onlines/someimage.png" />
<input type="radio" value="puppy"/>
</label>

And you can suppress all radio inputs like so:

input[type=radio] {
display: none;
}

It is important to note that the way that you group radio buttons together (i.e make it so clicking one de-clicks/de-selects the others) is by their ‚Äòname‚Äô property. Giving two radio buttons the same name groups them together.

For example, these two radio buttons could both be selected at the same time (you can have a favorite dog and a favorite cat):

<label>
<img src="http://onlines/someimage.png" />
<input type="radio" name="favorite_dog" value="dalmation"/>
</label>
<label>
<img src="http://onlines/someimage.png" />
<input type="radio" name="favorite_cat" value="tiger"/>
</label>

While these two radio buttons will automatically unselect one when the other is clicked (you cannot have two favorite dogs):

<label>
<img src="http://onlines/someimage.png" />
<input type="radio" name="favorite_dog" value="dalmation"/>
</label>
<label>
<img src="http://onlines/someimage.png" />
<input type="radio" name="favorite_dog" value="retriever"/>
</label>

Choose Your Own Adventure Part and Git Feature Flow

You are required to choose 1 of the next 2 options to complete. Extra credit if you do both, but 1 is part of the assignment.

We‚Äôre going to start learning how to use github branches and pull requests for something called git feature flow.

Git Feature Flow is a branching strategy that helps teams manage feature development in a structured way. We aren‚Äôt yet working in a team, but we‚Äôre going to start using feature branches to keep track of our work.

In Git Feature Flow, you would normally:

  • git checkout -b my_new_feature create a new branch
  • work on branch, git commit, etc
  • git push origin my_new_feature push your new branch to github
  • go to github and create pull request comparing main branch to my_new_feature
  • (optionally) tag someone to review
  • (once they approve) - go back on GitHub and merge it in

This workflow helps keep the main branch stable while allowing parallel feature development. It also provides a nice history for your project with a distinct timeline for when certain features were added.
